# P01 — v5.4 Product Data Contract, Dashboard Lifecycle, and Current-Run Provenance

## Purpose
Freeze one normalized data contract so domain engines, dashboard, screenshots, validators, docs, and manifests reference the same current-run evidence.

## Preconditions
- All earlier phases are PASS in `docs/v5_4_build_state.json`.
- `python scripts/validate_v5_4_state.py` passes before edits.
- Read `AGENTS.md`, `CODEX_HYNIX_V5_4_FINAL_PRODUCTIZATION_MASTER.md`, the source directive, contract lock, baseline audit, and this ticket completely.
- Modify the current product in place. Do not create a nested clone.

## Assigned agents
- Writer: `product-contract-engineer`
- Independent reviewer: `product-contract-reviewer`
- Writer cannot approve the phase or edit authoritative state to PASS. Parent closes P0/P1 findings and updates state.

## Dashboard change
All nine pages consume normalized domain objects. Missing mandatory arrays become an ERROR state with evidence path; JavaScript may not synthesize data.

## Implementation tickets
1. Move dashboard generation out of the monolithic scenario runner into domain-specific pure builders.
2. Group public-tool raw/parsed/receipt artifacts into one tool record; do not render duplicates.
3. Require source path, SHA-256, run ID, producer, and freshness for every top-level page object.
4. Copy generated dashboard data to frontend only after schema and provenance validation.
5. Make producer order explicit: engines/tools → dashboard bundle → UI → screenshots.

## Required tests
- Byte-for-byte deterministic bundle for fixed inputs.
- Stale run ID or source hash mismatch fails.
- Missing factory nodes or Fab trajectory is an ERROR, not fallback data.
- One tool produces exactly one dashboard record.

## Validation commands
```bash
pytest -q tests/test_dashboard_data_contract.py
python scripts/generate_dashboard_data.py --strict
python validation/validate_dashboard_data_v5_4.py --strict
```

## Acceptance / Done criteria
- Dashboard schema is frozen before P02–P05 parallel work.
- No frontend-generated domain evidence remains.
- All required page objects have lineage.

## Forbidden shortcuts
- Copying scenario YAML metrics directly into dashboard
- Duplicate tool cards
- Silent default data
- Changing shared schema in child worktrees without parent integration

## Handoff evidence
Writer returns JSON conforming to `.codex/v5_4/schemas/phase_handoff.schema.json`: changed files, implemented symbols, exact commands and return codes, current-run artifact paths/hashes/run IDs, dashboard impact, open risks, and reviewer request. Reviewer returns severity-ranked findings and explicit PASS/FAIL. Parent fixes every P0/P1, reruns the commands and reviewer, then updates authoritative state.
